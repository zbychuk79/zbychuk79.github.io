Docker - debian-wrt image

Przedstawię teraz jak utworzyć środowisko przydatne do generowania obrazów firmware OpenWRT dla moich punktów WiFi działających na SBC Raspberry Pi. Pozwala ono również na kompilowanie własnej wersji iPXE.

Najpierw pobieramy potrzebne nam pliki:

git clone https://github.com/zbychuk79/docker-wrt.git

W katalogu docker-wrt mamy następujące pliki:

  • Dockerfile - plik sterujący dla Docker‘a
  • wrt_script - uruchamiany podczas tworzenia obrazu
  • docker-image - generowanie obrazu Debian‘a z potrzebnymi programami i ustawieniami
  • docker-container - utworzenie kontenera na podstawie wygenerowanego wcześniej obrazu
  • docker-bash - uruchomienie bash w naszym nowym kontenerze

Po uruchomieniu ostatnich trzech skryptów, można wykonywać polecenia w naszym nowym kontenerze. W /usr/local/bin są skrypty wrt i ipxe, którymi możemy się posłużyć.

Inne przydatne skrypty:

  • docker-clean - kasowanie wygenerowanego powyżej kontenera i obrazu docker-a
