Docker - debian-wrt image

Przedstawię teraz jak utworzyć środowisko przydatne do generowania obrazów systemu OpenWRT dla moich punktów WiFi działających na SBC Raspberry Pi. Pozwala to też na kompilowanie własnej wersji iPXE.

Najpierw tworzymy katalog, gdzie bedą wszystkie potrzebne nam pliki (np. debian-wrt). W tym katalogu tworzymy następujące pliki:

  • Dockerfile - konfiguracja obrazu dla Docker‘a
FROM debian:latest
COPY wrt_script /
RUN /wrt_script
  • wrt_script - uruchamiany podczas tworzenia obrazu
#!/bin/bash

apt update
apt upgrade -y
apt install -y mc gawk python aptitude links wget curl build-essential genisoimage \
 libncurses5-dev zlib1g-dev isolinux git gettext libssl-dev xsltproc liblzma-dev \
 dnsutils

serwer="https://zbychuk79.github.io"

cd /usr/local/bin
wget $serwer/openwrt/wrt
wget $serwer/ipxe/ipxe
chmod +x *

cd /
wget $serwer/docker-wrt/.config/mc.tar.gz
mkdir -p /root/.config
cd /root/.config
tar xvfz /mc.tar.gz
rm /mc.tar.gz
  • docker-build-wrt-image - generowanie obrazu Debian‘a z potrzebnymi programami i ustawieniami
#!/bin/bash
sudo docker build -t debian-wrt .
  • docker-wrt-container - utworzenie kontenera na podstawie wygenerowanego wcześniej obrazu
#!/bin/bash
sudo docker run -d -t --name wrt debian-wrt
  • docker-wrt-bash - uruchomienie bash w naszym nowym kontenerze
#!/bin/bash
sudo docker exec -it wrt bash

Teraz można wykonywać polecenia w naszym nowym kontenerze. W /usr/local/bin są skrypty wrt i ipxe, którymi możemy się posłużyć.

Inne przydatne skrypty:

  • docker-rm-ex-c - kasowanie ostatnio uruchomionych i zatrzymanych kontenerów
#!/bin/bash
sudo docker ps -a | grep Exit | cut -d ' ' -f 1 | xargs sudo docker rm
